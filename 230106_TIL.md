# 디지털 서비스 기획

1. 기초 점검
    - 정의를 명확하게 아는 것이 중요.
        - 디지털화
            - 오프라인에 있는 것을 온라인으로 가져오는 것
        - 디지털전환(Digital Transformation)=DX
            - 디지털 기술이 제공하는 기회를 포착할 수 있도록 조직의 전략과 구조를 조정하는 것
    - 고객이 관심을 가지면 상품 제공자보다 더 많은 정보를 얻을 수 있어 서비스/제품에 대한 만족의 잣대 역시 훨씬 높아지고 까다로워졌음
        - 시대의 흐름에 따라 트렌드가 빠르게 바뀜
    - 서비스 기획
        - 고객이 선택해 사용하며 차별화된 가치를 느끼고 조직은 목표 성과를 달성할 수 있도록 필요한 ‘지도’를 만드는 활동
    - 서비스 제공 관점 변화
        - ‘어떻게 하면 고객 경험과 관계를 강화할 수 있을까’로 관점 변경 필요
    - 경제적 가치 발전 : 서비스의 전달을넘어서 고객 경험의 연출이 필요
        - 디자인 씽킹 : 공감에 기반한 고객 니즈와 관련한 혁신의 필요성에서 출발 → 빠른 실행과 경험의 중요성을 강조
2. 디지털 서비스 구현 관점의 디자인 이해
    - 스타트업의 4개의 기둥
    1. LEAN 방법론
    2. 디자인 씽킹
    3. Customer Development
    4. Agile software development
    - 뱅뱅이론
        - 우리가 아는 사실은 전체 관점에서 거짓일 가능성이 있다.
    - 뱅뱅이론에 의거해서 제공자 입장에서 고객 관점으로 이동한다.
        - 우리가 속한 영역과 고객의 정규 분포 중앙과 다를 수 있다는 점 인지.
- UX
    - 사용자 경험을 의미
    - 세 가지 특징
        1. 총체성
            - 모든 것들의 합
        2. 인간중심적
        3. 전략적
- CX
    - 소비자 경험을 의미
- SX
    - 해당 산업에 집중
1. 디지털 서비스 기획 개발자 관점의 디지털 서비스 구현 프로세스
    - 방향의 제시
        - 기획 업무 운영의 기본
        - 6가지 체크리스트 작성
    - 구현의 제안
        - 서비스 제작 기본 과정 및 진행 순서
        - 단계별로 정의된 일반적인 방법론이 존재하며 상황에 맞게 응용 후 사용
2. 디지털 서비스 개발자 주도의 프로젝트 방법론 기획 포인트
    - 협업을 위한 프로젝트 방법론
        - Waterfall Approach
            - 이전 단계에 대한 승인 후 다음 단계로 넘어가는 방법
            - 대기업, 장기 SI 프로젝트
            - 성공 사례를 찾기 쉽다
        - Agile Approach
            - 개발과 디자인 등의 과정을 여러 개로 나눠서 단계 별 프로세스를 반복해 모든 과정을 완료
            - 스타트업, TF형태의 프로젝트
            - 성공 사례를 찾기 어려워서 비판 받는다.
            - 그럴지만 고객 개발과 유사하다는 점 때문에 많이 사용한다.
3. 협업 측면의 프로세스 주요 활동
    - 고객 인사이트 중심의 조사 활동 핵심은 (참여)관찰과 (심층)인터뷰
        - 한 명이라도 만나면 문제의 25% 확인
        - 극단적 특성을 가진 사람들을 정성 조사할 때 혁신의 기회가 존재
    - 프로토타이핑
        - 실제 테스트를 수행하면서도 쉽게 피드백 가능
        - 설계뿐만 아니라 사고 과정 자체에도 도움을 줌
- [https://opentutorials.org/course/1688](https://opentutorials.org/course/1688) 추천
- 트렌드 코리아 등의 자료를 계속적으로 읽는 것 중요
    - 체리슈머(실속형 소비자) 등 신조어가 계속 등장
    - 개발에 고여있기보다 IT산업 전체를 계속 이해하는 습관 필요
- 벤치마킹을 급히 해버리면 Pre-Cursor(레퍼런스에서 장점을 잘 선택해서 가져오는 것)이 아닌 Copycat(비판적인 사고 없이 그대로 배낀 것)이 된다.
- TIP
    - 프로젝트에서 고객 인터뷰(1~2명) 추천
    - 내 위치와 고객 위치의 정규분포도를 비교해서 내 생각과 평균의 생각이 다를 수 있다는 점 인지
    - 프로젝트 내에서 6개월 뒤에 진행사항을 적어 가능성을 나타내자
    - 레퍼런스를 할 때 같은 성격을 가진 다른 산업을 보자(Copycat방지)
    - 보험,카드 ↔ 금융 ↔ 제약, 뷰티
